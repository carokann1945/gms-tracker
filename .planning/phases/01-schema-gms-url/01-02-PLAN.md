# Plan 01-02: buildGmsUrl 구현 및 파이프라인 통합

## Goal

`src/parser.js`에 `buildGmsUrl(id, name)` 함수를 추가하고, `index.js`의 row 객체에 `gms_url` 필드를 포함시켜 Supabase upsert 시 저장되도록 한다.

## Why This Plan Exists

- 이벤트 상세 페이지 URL(`https://www.nexon.com/maplestory/news/events/{id}/{slug}`)을 DB에 저장하려면 파이프라인에서 URL을 생성해야 한다.
- slug는 `name` 필드의 순수 문자열 변환으로 생성 가능하다 — 외부 라이브러리 불필요.
- `parser.js`는 이미 텍스트 변환 함수(`extractBodyText`, `extractBodyImageUrls`)를 소유하는 모듈이다. `buildGmsUrl`은 여기 속한다.

## Tasks

### T1: `src/parser.js`에 `buildGmsUrl` 추가

파일 **하단**에 다음 named export를 추가한다:

```javascript
/**
 * GMS 이벤트 상세 페이지 URL을 생성한다.
 * @param {string|number} id
 * @param {string} name
 * @returns {string}
 */
export function buildGmsUrl(id, name) {
  const slug = name
    .replace(/[^a-zA-Z0-9 ]/g, ' ') // non-alphanumeric (except space) → space
    .toLowerCase()
    .trim()
    .replace(/\s+/g, '-')            // runs of whitespace → single hyphen
    .replace(/-+/g, '-')             // collapse consecutive hyphens
    .replace(/^-|-$/g, '');          // strip leading/trailing hyphens

  return `https://www.nexon.com/maplestory/news/events/${id}/${slug}`;
}
```

**검증 예시 (4개 모두 통과해야 함):**

| Input `name` | Expected slug |
|---|---|
| `"Monster Park Mayhem!"` | `monster-park-mayhem` |
| `"Hero & Centichoro Punch King Coin Shops"` | `hero-centichoro-punch-king-coin-shops` |
| `"[Update Feb 19] February 2026 Hot Weeks!"` | `update-feb-19-february-2026-hot-weeks` |
| `"Goodbye, Victoria Cup!"` | `goodbye-victoria-cup` |

### T2: `index.js` import 구문 수정

기존 import에 `buildGmsUrl`을 추가한다:

```javascript
// Before
import { extractBodyImageUrls, extractBodyText } from './src/parser.js';

// After
import { extractBodyImageUrls, extractBodyText, buildGmsUrl } from './src/parser.js';
```

### T3: `index.js` rows.push() 블록에 `gms_url` 추가

`rows.push(...)` 블록에서 `gms_url` 필드를 추가한다. name 필드와 동일한 null-safe 패턴을 사용한다:

```javascript
// Refactored pattern: extract eventName once to avoid divergence
const eventName = detail.name ?? detail.title ?? '';
rows.push({
  id,
  name: eventName,
  image_url: detail.imageThumbnail ?? null,
  event_period,
  gms_url: buildGmsUrl(id, eventName),
});
```

**핵심:** `buildGmsUrl`은 `name` 필드와 동일한 `detail.name ?? detail.title ?? ''` 표현식을 사용해야 한다. 다른 필드를 사용하면 이름 불일치가 발생한다.

## Constraints

- `kms_url` 필드는 row 객체에 **절대 포함하지 않음**: omit해야 Phase 2에서 채운 값이 덮어쓰이지 않는다 (`undefined`도 금지)
- `db.js` 수정 금지: DB 레이어는 thin persistence wrapper로 유지
- 새 파일 생성 금지: `buildGmsUrl`은 `src/parser.js`에 추가 (별도 모듈 과잉설계)
- slug 라이브러리(`slugify` 등) 설치 금지: 순수 JS 5줄로 충분
- 기존 throttle(500ms), idempotency(upsert onConflict: 'id'), try-catch 구조 변경 금지

## Execution Order

1. T1: `src/parser.js` 수정 (buildGmsUrl 추가)
2. T2 + T3: `index.js` 수정 (import 및 rows.push 동시 수정)
