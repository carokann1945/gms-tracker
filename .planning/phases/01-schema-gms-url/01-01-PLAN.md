# Plan 01-01: Supabase Schema Migration

## Goal

`events` 테이블에 `gms_url`(text, nullable)과 `kms_url`(text, nullable) 컬럼을 추가한다.
이 마이그레이션은 코드 변경 없이 Supabase Dashboard SQL Editor에서 한 번만 실행하는 DDL 작업이다.

## Why This Plan Exists

- `gms_url` 컬럼이 없으면 01-02의 코드 변경 후 upsert가 `column "gms_url" does not exist` 에러로 실패한다.
- 이 프로젝트에는 마이그레이션 프레임워크(Prisma, Flyway, Supabase CLI)가 없다. 수동 DDL이 유일한 방법이다.
- `kms_url`은 Phase 2에서 채워지지만, 두 컬럼을 지금 같이 추가하면 Phase 2에서 별도 마이그레이션이 불필요하다.

## Tasks

### T1: SQL Editor에서 마이그레이션 실행

**실행 위치:** Supabase Dashboard → SQL Editor

```sql
-- Safe to re-run (IF NOT EXISTS)
ALTER TABLE events
  ADD COLUMN IF NOT EXISTS gms_url text,
  ADD COLUMN IF NOT EXISTS kms_url text;
```

**실행 순서:**
1. Supabase Dashboard → 프로젝트 선택 → SQL Editor
2. 위 SQL을 붙여넣기 후 Run
3. 성공 메시지(`ALTER TABLE`) 확인

### T2: 컬럼 추가 검증

마이그레이션 완료 후 Table Editor 또는 아래 쿼리로 컬럼 존재 확인:

```sql
SELECT column_name, data_type, is_nullable
FROM information_schema.columns
WHERE table_name = 'events'
  AND column_name IN ('gms_url', 'kms_url');
```

**기대 결과:** 두 행이 반환되고, `data_type = 'text'`, `is_nullable = 'YES'`

## Constraints

- `NOT NULL` 제약 추가 금지: 기존 행에 기본값이 없으므로 nullable로 유지
- 코드 파일 변경 없음: 이 Plan은 순수 DDL 작업
- **01-02보다 먼저 실행해야 함**: 컬럼이 없는 상태에서 코드를 실행하면 upsert 실패

## Verification

- [ ] `gms_url` 컬럼이 `events` 테이블에 존재
- [ ] `kms_url` 컬럼이 `events` 테이블에 존재
- [ ] 두 컬럼 모두 `text` 타입, nullable
- [ ] SQL을 다시 실행해도 에러 없음 (IF NOT EXISTS 확인)
