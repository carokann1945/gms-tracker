# Plan 02-01: fetchKmsEventList() 구현

## Goal

`src/fetcher.js`에 `fetchKmsEventList()` 함수를 추가한다. 이 함수는 KMS 이벤트 페이지(`maplestory.nexon.com/News/Event`)를 스크래핑하여 전체 이벤트 목록(`[{ id, name }]`)을 반환한다. Phase 2의 매처(02-02)와 파이프라인 통합(02-03)이 이 함수의 반환값을 사용한다.

## Why This Plan Exists

- KMS URL 매칭을 위해서는 KMS 이벤트 전체 목록이 필요하다. 공식 API가 없으므로 HTML 스크래핑이 유일한 방법이다.
- `cheerio`는 이미 v1.2.0으로 설치되어 있고, `fetch`는 Node.js 내장 API로 신규 의존성이 불필요하다.
- `src/fetcher.js`는 이미 GMS API 페칭 로직을 소유하는 모듈이다. KMS 스크래퍼를 여기에 추가하면 자연스럽게 역할이 확장된다.

## Tasks

### T1: `fetchKmsPage` 헬퍼 추가 (private, export 없음)

`src/fetcher.js` 내부에 다음 헬퍼를 추가한다:

```javascript
async function fetchKmsPage(url) {
  const res = await fetch(url, {
    headers: {
      'User-Agent': 'Mozilla/5.0 (compatible; gms-tracker/1.0)',
      'Accept': 'text/html,application/xhtml+xml',
    },
  });
  if (!res.ok) throw new Error(`KMS fetch failed: ${res.status} ${res.statusText} (${url})`);
  return res.text();
}
```

**중요:** `User-Agent` 헤더 없이 요청하면 Nexon CDN/WAF가 빈 HTML을 반환할 수 있다 (Pitfall 6).

### T2: `parseOngoingEvents` 헬퍼 추가 (private)

```javascript
function parseOngoingEvents(html) {
  const $ = load(html);
  const events = [];
  $('dt a[href]').each((i, el) => {
    const href = $(el).attr('href');
    const match = href?.match(/^\/News\/Event\/(\d+)$/);
    if (!match) return;
    const title = $(el).text().replace(/\s+/g, ' ').trim();
    if (title) events.push({ id: match[1], name: title });
  });
  return events;
}
```

**셀렉터 근거:** `/^\/News\/Event\/(\d+)$/` 패턴은 정확히 Ongoing 이벤트 href만 매칭한다. `/News/Event/Closed/{id}` 및 기타 내비게이션 링크는 제외된다. 라이브 데이터로 21개 이벤트 추출 확인됨 (2026-03-27).

### T3: `parseClosedEvents` 헬퍼 추가 (private)

```javascript
function parseClosedEvents(html) {
  const $ = load(html);
  const events = [];
  $('dd.data').each((i, el) => {
    const link = $(el).find('a[href*="/News/Event/Closed/"]').first();
    const href = link.attr('href');
    const match = href?.match(/\/News\/Event\/Closed\/(\d+)/);
    if (!match) return;
    const title = $(el).find('em.event_listMt').text().replace(/\s+/g, ' ').trim();
    if (title) events.push({ id: match[1], name: title });
  });
  return events;
}
```

**셀렉터 근거:** `dd.data em.event_listMt`는 사이드바 Ongoing 이벤트를 포함하지 않는다. `dt a` 셀렉터는 Closed 페이지의 사이드바 이벤트(21개)를 혼합시키므로 사용 금지 (Pitfall 1). 라이브 데이터로 12개/페이지 확인됨.

### T4: `fetchKmsEventList` export 추가

```javascript
export async function fetchKmsEventList() {
  const events = [];
  try {
    // 1. Ongoing events (단일 페이지 — 페이지네이션 불필요)
    const ongoingHtml = await fetchKmsPage(
      'https://maplestory.nexon.com/News/Event/Ongoing'
    );
    events.push(...parseOngoingEvents(ongoingHtml));
    console.log(`[fetcher] KMS ongoing: ${events.length} events`);

    // 2. Closed events (빈 페이지 감지 시 중단)
    let page = 1;
    while (true) {
      await sleep(500); // Nexon 서버 보호용 throttle
      const html = await fetchKmsPage(
        `https://maplestory.nexon.com/News/Event/Closed?page=${page}`
      );
      const pageEvents = parseClosedEvents(html);
      if (pageEvents.length === 0) break;
      events.push(...pageEvents);
      console.log(`[fetcher] KMS closed page ${page}: ${pageEvents.length} events`);
      page++;
    }
  } catch (err) {
    console.error('[fetcher] fetchKmsEventList error:', err.message);
    // 에러 이전까지 수집된 이벤트 반환 (graceful degradation)
  }

  console.log(`[fetcher] KMS total: ${events.length} events`);
  return events;
}
```

**중단 조건:** `parseClosedEvents(html).length === 0` — 빈 페이지 감지로 중단. 총 페이지 수는 HTML에 노출되지 않으므로 이 방식이 유일하게 안정적인 방법이다. 라이브 검증: page 99 유효, page 100 빈 페이지 (2026-03-27).

## Constraints

- `Promise.all`로 KMS 페이지를 병렬 요청하는 것 금지 — CLAUDE.md에서 외부 API Rate Limit 초과 방지를 명시
- `src/fetcher.js` 외의 파일 수정 금지 — 이 plan의 범위는 fetcher.js 단독
- 새 파일 생성 금지 — 기존 `src/fetcher.js`에 추가
- `kms_url` 최종 URL은 `/News/Event/{id}` 형식이어야 함 — `/Closed/{id}` 경로 저장 금지 (Pitfall 3, ID만 숫자 추출로 해결됨)

## Anti-Patterns

- **Pitfall 1 방지:** Closed 페이지에서 `dt a` 셀렉터 사용 금지 → `dd.data em.event_listMt` 사용
- **Pitfall 6 방지:** `User-Agent` 헤더 누락 금지
- **Pitfall 3 방지:** `href`에서 ID 숫자만 추출, URL 재구성 시 `/News/Event/{id}` 형식 사용
