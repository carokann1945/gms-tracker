# Plan 02-03: index.js 파이프라인 통합

## Goal

`index.js`에 `fetchKmsEventList()`와 `findKmsUrl()` 호출을 통합하여, 각 신규 이벤트의 `kms_url`을 Supabase에 저장하도록 한다. `fetchKmsEventList()`는 루프 전 1회만 호출하고, `findKmsUrl(eventName, kmsList)`는 각 이벤트마다 호출한다.

## Why This Plan Exists

- 02-01과 02-02의 함수를 파이프라인에 연결하는 마지막 단계이다.
- `fetchKmsEventList()`를 루프 내부에서 호출하면 이벤트 수 × ~100페이지 스크래핑이 발생한다 (Pitfall 5). 반드시 루프 외부에서 1회만 호출해야 한다.
- 신규 이벤트가 없으면 KMS 스크래핑 자체가 불필요하다. 조기 반환 직후가 아닌, `newItems`가 존재함이 확인된 이후에 `fetchKmsEventList()`를 호출해야 한다.

## Tasks

### T1: import 구문 추가

`index.js` 최상단 import에 다음을 추가한다:

```javascript
// Before
import { fetchNewsList, fetchEventDetail, sleep } from './src/fetcher.js';

// After
import { fetchNewsList, fetchEventDetail, sleep, fetchKmsEventList } from './src/fetcher.js';
```

그리고 matcher import를 추가한다:

```javascript
import { findKmsUrl } from './src/matcher.js';
```

### T2: `fetchKmsEventList()` 호출 위치 결정

**올바른 위치:** `newItems`가 비었는지 확인 후 조기 반환하는 블록 **이후**, 4단계 루프 **이전**.

```javascript
  // newItems 존재 확인 완료 후 — KMS 스크래핑은 처리할 이벤트가 있을 때만
  console.log(`[main] ${newItems.length} new item(s) to process`);

  // ★ KMS 이벤트 목록을 루프 전 1회 로드
  const kmsList = await fetchKmsEventList();

  // 3. 신규 항목 상세 API 순차 호출 (0.5초 throttle)
  ...
```

**잘못된 위치 (금지):**
- `for (const detail of newDetails)` 루프 내부 — 이벤트 수 × 전체 스크래핑 발생
- `newItems.length` 확인 전 — 처리할 이벤트가 없어도 KMS를 스크래핑하게 됨

### T3: `findKmsUrl()` 호출 및 `kms_url` 추가

4단계 루프(`for (const detail of newDetails)`) 내부에서 `event_period` 결정 이후, `rows.push()` 직전에 추가한다:

```javascript
    console.log(`[main] id=${id} → period="${event_period ?? 'not found'}"`);

    const eventName = detail.name ?? detail.title ?? '';
    const kms_url = await findKmsUrl(eventName, kmsList);  // ★ 추가

    rows.push({
      id,
      name: eventName,
      image_url: detail.imageThumbnail ?? null,
      event_period,
      gms_url: buildGmsUrl(id, eventName),
      kms_url,  // ★ 추가
    });
```

**변경 후 전체 4단계 루프:**

```javascript
  // 4. 각 이벤트마다 다중 계층 추출로 날짜 파싱
  const rows = [];
  for (const detail of newDetails) {
    const id = String(detail.id);
    const bodyHtml = detail.body ?? '';
    let event_period = null;

    // 1계층: HTML 텍스트 → AI 정제 1차 시도
    const bodyText = extractBodyText(bodyHtml);
    event_period = await extractEventPeriodWithAI(bodyText);

    if (!event_period) {
      // 2계층: 이미지 상위 2장 OCR 텍스트를 합쳐서 AI에 1회만 질의
      const candidates = extractBodyImageUrls(bodyHtml).slice(0, OCR_LIMIT);
      const ocrTexts = [];
      for (const imageUrl of candidates) {
        const ocrText = await extractTextFromImage(imageUrl);
        if (ocrText) ocrTexts.push(ocrText);
      }
      if (ocrTexts.length) {
        event_period = await extractEventPeriodWithAI(ocrTexts.join('\n\n'));
      }
    }

    console.log(`[main] id=${id} → period="${event_period ?? 'not found'}"`);

    const eventName = detail.name ?? detail.title ?? '';
    const kms_url = await findKmsUrl(eventName, kmsList);

    rows.push({
      id,
      name: eventName,
      image_url: detail.imageThumbnail ?? null,
      event_period,
      gms_url: buildGmsUrl(id, eventName),
      kms_url,
    });
  }
```

## Constraints

- `fetchKmsEventList()` 루프 내부 호출 금지 (Pitfall 5)
- `kms_url: undefined` 또는 `kms_url: null` 명시 없이 row 객체에서 omit 금지 — `findKmsUrl`이 `null`을 반환하므로 `kms_url: null`이 upsert에 포함된다 (정상 동작)
- 기존 throttle(500ms), idempotency(upsert onConflict: 'id'), try-catch 구조 변경 금지
- `db.js`, `parser.js`, `ai.js`, `ocr.js` 수정 금지 — 이 plan의 범위는 `index.js` 단독

## Execution Order

1. T1: import 구문 추가
2. T2: `kmsList` 로드 위치 결정 및 코드 삽입
3. T3: `kms_url` 필드를 루프 내부와 `rows.push()`에 추가
